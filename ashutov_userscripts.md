### [ Сценарий 1 - Регистрация пользователя ]
1. Пользователь нажимает кнопку Войти (всегда есть в правом верхнем углу страницы, иногда появляется и в других элементах сайта)
и автоматически перенаправляется на страницу авторизации и регистрации, и под этой формой он нажимает кнопку "Зарегистрироваться".
2. Пользователь вводит адрес электронной почты, который будет использоваться в системе.
3. Если на данный адрес электронной почты уже существует аккаунт, пользователю сообщится об этом и ему предложат войти с помощью него.
4. Далее пользователь вводит логин, который будет идентифицировать его в системе.
5. Если выбранный логин уже существует в системе, то пользователю сообщается об этом и предлагается придумать новый логин.
6. Пользователь вводит пароль, с которым он будет заходить в систему.
7. Если пароль содержит менее 8 символов или в нём отсутствуют специальные символы/цифры, то система сообщает, что
пароль должен быть от 8 символов в длину, пользователь должен придумать новый пароль.
8. Если введённый адрес электронной почты не соответствует формату или уже зарегистрирован в системе, то она выводит
сообщение об ошибке пользователю и просит его ввести подходящий адрес.
9. Если все введённые данные соответствуют требованиям регистрации, то пользователь успешно регистрируется и перенаправляется обратно на сайт Fundupy.

### [ Сценарий 2 - Авторизация пользователя ]
1. Пользователь нажимает кнопку Войти (всегда есть в правом верхнем углу страницы, иногда появляется и в других элементах сайта)
и автоматически перенаправляется на страницу авторизации и регистрации.
2. Если пользователь был авторизован ранее (в течение часа), то его автоматически вернёт на сайт Fundupy.
3. В ином от пункта 2 случае пользователя попросят ввести логин и пароль.
4. После введения данных пользователь нажимает кнопку "Войти".
5. Если введённый адрес электронной почты не зарегистрирован в системе, то она выводит
сообщение об ошибке пользователю и просит его ввести верный адрес.
6. Если введённый пароль не соответствует зарегистрированным в системе данным, то она выводит
сообщение об ошибке пользователю и попросит попробовать ввести данные заново.
7. После успешной авторизации пользователя перенаправляют на сайт Fundupy.

### [ Сценарий 3 - Поиск и просмотр страницы Автора ]
1. Пользователь находит строку поиска, которая расположена в верхней части страницы.
2. Пользователь нажимает по ней левой кнопкой мыши и вводит интересующий его запрос.
3. При нажатии кнопки Лупы справа от строки поиска или кнопки Enter на клавиатуре производится переход на страницу с результатами, которые
обнаружила система на основе запроса пользователя. Это могут быть Авторы или Проекты.
4. Пользователь может пролистать все результаты поиска с помощью колёсика мыши.
5. Пользователь может нажать по одному из результатов поиска и при нажатии левой кнопкой мыши по нему перейти на страницу Автора.
6. На странице Автора есть вкладки "Активность", "Посты", "Проекты", "О себе", на каждой отображается соответствующая информация.
7. Если результат не удовлетворил пользователя, он может вернуться к результатам поиска и продолжить просмотр других страниц Авторов и Проектов.

3. При нажатии кнопки Лупы справа от строки поиска или кнопки Enter на клавиатуре производится переход на страницу с результатами, которые
обнаружила система на основе запроса пользователя. Это могут быть Авторы или Проекты.
4. Пользователь может пролистать все результаты поиска с помощью колёсика мыши.
5. Пользователь может нажать по одному из результатов поиска и при нажатии левой кнопкой мыши по нему перейти на страницу Проекта.
6. На странице Проекта есть вкладки "Активность", "Обновления", "Комментарии", на каждой отображается соответствующая информация
7. Если результат не удовлетворил пользователя, он может вернуться к результатам поиска и продолжить просмотр других страниц Авторов и Проектов.

### [ Сценарий 5 - Регистрация первой страницы автора ]
1. Если пользователь авторизован, то он может создать несколько страниц Автора. Если пользователь не зарегистрирован или не авторизован, то смотреть Сценарии 1 и 2.
2. В меню аккаунта, которое расположено в правом верхнем углу экрана, Пользователь нажимает кнопку "Стать Автором".
3. После этого система перенаправляет пользователя на форму создания страницы Автора.
4. Пользователь вводит имя Автора (иначе, Никнейм), который будет идентифицировать его в системе. Оно должно быть уникальным для каждого Автора.
5. Если введённый Никнейм уже занят, то пользователю сообщается об этом и предлагается придумать новый.
6. По желанию пользователь может заполнить поля "Описание", "Аватар", "Отображаемое имя Автора" и ссылки на другие ресурсы.
7. После заполнения данных Пользователь нажимает кнопку "Создать автора".
8. После успешной регистрации система перенаправит пользователя на его только что созданную страницу.

### [ Сценарий 6 - Регистрация дополнительной страницы автора ]
1. Если пользователь авторизован, то он может создать несколько страниц Автора. Если пользователь не зарегистрирован или не авторизован, то смотреть Сценарии 1 и 2.
2. В меню аккаунта, которое расположено в правом верхнем углу экрана, Пользователь нажимает кнопку "Добавить Автора".
3. После этого система перенаправляет пользователя на форму создания страницы Автора.
4. Пользователь вводит имя Автора (иначе, Никнейм), который будет идентифицировать его в системе. Оно должно быть уникальным для каждого Автора.
5. Если введённый Никнейм уже занят, то пользователю сообщается об этом и предлагается придумать новый.
6. По желанию пользователь может заполнить поля "Описание", "Аватар", "Отображаемое имя Автора" и ссылки на другие ресурсы.
7. После заполнения данных Пользователь нажимает кнопку "Создать автора".
8. После успешной регистрации система перенаправит пользователя на его только что созданную страницу.

### [ Сценарий 7 - Публикация поста Автора ]
1. Если пользователь авторизован, и он владеет хотя бы одной страницей Автора, то он может написать и опубликовать Пост от имени Автора.
Если пользователь не зарегистрирован или не авторизован, то смотреть Сценарии 1 и 2. Если пользователь не владеет страницей Автора, смотреть Сценарий 5.
2. На странице Автора во вкладке "Посты" пользователь, владеющий этим Автором, может нажать кнопку "Написать пост", которая находится под меню выбора вкладок.
3. В открывшейся форме пользователь должен указать содержание поста (Основное содержимое) в текстовом поле и по желанию может указать заголовок Поста.
4. Текст поддерживает язык разметки [Markdown](https://www.markdownguide.org/basic-syntax/), так что при желании пользователь может стилизировать свой текст. 
5. Перед публикацией Поста Пользователь может увидеть, как будет выглядеть Пост, перейдя во вкладку Предпросмотр выше формы.
6. Пользователь может опубликовать пост или отменить публикацию с помощью нажатия по соответствующей кнопке внизу формы.
7. После нажатия кнопки "Опубликовать" пост появится на странице Автора и будет доступен всем пользователям.

### [ Сценарий 8 - Создание страницы Проекта ]
1. Если пользователь авторизован, и он владеет хотя бы одной страницей Автора, то он может создать страницу Проекта для этого Автора.
Если пользователь не зарегистрирован или не авторизован, то смотреть Сценарии 1 и 2. Если пользователь не владеет страницей Автора, смотреть Сценарий 5.
2. На странице Автора во вкладке "Проекты" пользователь может нажать кнопку "Создать проект".
3. В открывшейся форме пользователь должен указать Название проекта, его идентификатор, может добавить описание и обложку, презентующую этот Проект,
в форматах PNG, JPEG, WEBP или не анимированный GIF.
4. Перед публикацией Проекта Пользователь может увидеть, как будет выглядеть страница Проекта, перейдя во вкладку Предпросмотр вверху страницы.
5. Пользователь может подтвердить создание или отменить процесс с помощью нажатия по соответствующей кнопке внизу формы
6. После нажатия кнопки "Создать проект" Пользователя перенаправит на новую страницу Проекта, Проект появится на странице Автора 
во вкладке "Проекты" и будет доступен всем Пользователям для просмотра.
7. На данной странице Проекта Пользователь-владелец Автора может добавить Цели поддержки (целевую сумму сбора средств) и/или Награды поддержавшим пользователям.

### [ Сценарий 9 - Пополнение баланса и оформление поддержки ]
1. Если пользователь авторизован, то он может начать поддерживать Авторов и их Проекты. Если пользователь не зарегистрирован или не авторизован, то смотреть Сценарии 1 и 2.
2. В меню настроек, которое открывается с помощью меню аккаунта в правом верхнем углу экрана, пользователь нажимает на поле Баланс.
3. Система перенаправляет Пользователя на страницу информации о Балансе и способах пополнения.
4. При нажатии кнопки "Пополнить баланс" пользователю будет предложено выбрать способ пополнения суммы.
5. После выбора способа пополнения Пользователь вводит сумму, на которую готов пополнить баланс аккаунта.
6. После определения пополняемой суммы Пользователь нажимает кнопку "Пополнить", после чего на баланс его аккаунта добавится эта сумма.
Так как проект носит демонстрационный характер, валюта также является демонстрационной и не имеет никакой стоимости.
7. Теперь Пользователь, перейдя на страницу Проекта, который он хочет поддержать, может пожертвовать сумму, не превышающую его баланс.
8. После получения поддержки Автор обязуется выполнить свои условия поощрения.
